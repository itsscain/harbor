Harbor is adding a Claude AI companion (the user's stated headline feature). Foundation shipped 2026-06-22:

- **Bring-your-own key.** Each household pastes its own Anthropic API key in **Settings → AI Companion**. Stored in a dedicated `ai_config` table (`household_id` pk, `anthropic_api_key`, `enabled`). **Security invariant: the key must NEVER reach the wall/kiosk** — `ai_config` is excluded from `kiosk_snapshot`, untouched by the anon kiosk RPCs, RLS is `household_is_mine` only, and the settings page reads back only a "key is set" boolean, never the raw key. AI calls happen server-side only.
- **Model: Haiku** (`claude-haiku-4-5-20251001`) for cost. Helper `lib/ai/anthropic.ts` (`server-only`): `getHouseholdAi()`, `haikuJson()` (forced single-tool structured output, max_tokens caps, cacheable system prompt, minimal context), `aiErrorMessage()`.
- **Shipped:** (1) **Pantry-aware meal generation** — parent adds on-hand ingredients on the **Pantry** page (`/app/pantry`, reuses `list_items` kind='pantry'); `generateMealPlan` builds dinners around the pantry, writes per-meal notes ("Uses … · Need …"), and auto-adds missing ingredients to the grocery list. (2) **Screensaver daily brief** — `ai_briefs` cache table + `/api/ai/brief` route (validates device_secret via service role, returns today's meals + a once-per-day cached Haiku brief; key stays server-side); the kiosk Screensaver shows tonight's dinner + the brief. Helpers: `haikuJson`, `haikuText`.
- **Also shipped (AI-central pass):** AI **chore suggestions** (age-aware from birthday; `suggestChores`), AI **family insights** on the Insights page (`generateInsight`, on-demand), and **per-day chores** (day-of-week picker → `createChore` days_of_week; kiosk `runsToday` respects it). The brief route now **claims today's slot atomically** (upsert ignoreDuplicates) so Haiku runs at most once/household/day even on failure/race/multi-device, and uses a **timezone-correct** event window (kiosk sends tz offset). `generateMealPlan` dedups AI meals by date.
- **Also shipped:** per-kid **AI profile** (`children.ai_profile` jsonb; `buildChildProfile` → summary/interests/motivators/encouragement) — built parent-side (one Haiku call), synced to the wall, and the kid's ChildView shows a personalized encouragement line **offline** (rotates daily, no per-tap cost). Card: `AiProfileCard` on the child page.
- **Shipped: "Hey Harbor" voice.** `components/kiosk/VoiceButton.tsx` (tap-to-talk + opt-in wake word via Web Speech; speaks replies via SpeechSynthesis; calls `kiosk.syncNow` after an action so new data pulls to the wall) → `/api/ai/command` (device-validated like the brief route). Haiku **tool-use** maps the transcript to ONE whitelisted, household-scoped, **non-destructive** action: `add_to_grocery`, `add_chore` (this household's kids only), `plan_dinners`, or `reply`. Key server-side only; text capped 500; cross-household impossible; no deletes/settings/money. Future option if asked: PIN-gate voice actions so kids can't add chores/groceries.
- **Refactor:** pantry-aware planning lives in `lib/ai/mealPlan.ts` (`planDinners(supabase, household_id, key)`), reused by both `generateMealPlan` (parent) and the voice endpoint.
- **Shipped (non-AI, same push):** kiosk **routine screen overhaul** — see [[kiosk-professional-ui]] (whole-card tap targets, photo+progress header).
- **Shipped (non-AI):** kiosk **calendar event pop-out** (tap any event → detail overlay: emoji/title/time/child/person/location/recurrence, read-aloud, closes on X/backdrop/Esc; `EventDetail` in `CalendarView.tsx`). **Grounding v2 part 1 = custom "privileges lost" bubbles** (`groundings.privileges_lost` jsonb; `PrivilegesPicker` in GroundingCard; shown on the wall in ChildView + ChoresBoard reset block + read aloud; sanitized via `strList`).
- **Shipped: grounding v2 part 2 = House Rules + consequence ladder.** New `house_rules` table (`kind` 'rule'|'consequence', title/detail/emoji/sort_order; household-scoped RLS; rides the rebuilt `kiosk_snapshot`). Parent page `/app/rules` (add/edit/delete/reorder + one-tap seed); wall: `HouseRules.tsx` overlay opened from a Home tile (`onOpenHouseRules` in KioskShell), reads items aloud. The "escalation graph" was delivered as a **visual numbered consequence ladder** (no fragile timed automation). New household tables: add to `kiosk_snapshot` (rebuild the whole fn, mirror calm_tools) + `KioskSnapshot` + `applyPull` mergeById + a `set_updated_at` trigger; run `get_advisors` after.
- **Shipped: weekly chore rotation.** `chores.rotation_member_ids` (jsonb, rides to_jsonb); `lib/kiosk/chores.ts` `choreAssignee()` computes the weekly assignee offline (filters deleted members, falls back to anchor); wall shows the chore under the current kid with a 🔄 hint; parent toggle "Rotate between all the kids". **`rpc_kiosk_push` (migration 0025) must credit the assignee (anchor OR a `rotation_member_ids` member)** — the original child_id-only check silently dropped rotating completions.
- **Hey Harbor accuracy fixes (`/api/ai/command`):** now includes each kid's chores-for-today with done status (so it answers "what chores does X have"); routing is conservative — `reply()` is the default for any question, and add_to_grocery/add_chore/plan_dinners only fire on explicit commands (fixed "what's for dinner" wrongly bulk-adding groceries).
- **Shipped (non-AI):** per-child **Bedtime countdown** on the wall (visual sun→moon "time till bed" for pre-readers; `child.settings.bedtime`, no migration; `components/kiosk/BedtimeCountdown.tsx`, chip on Home + full on the kid's screen). Chore **day-picker** revert was visual only (saved fine) — fixed by keying the row form on `updated_at`.
- **Shipped:** **interactive screensaver** (`Screensaver.tsx`) — auto-flipping info-panel deck (AI brief, dinner, today, notes, countdowns, star leaders) from the local snapshot + brief; AI brief in its own panel with `**bold**`→`<strong>` (brief prompt forbids markdown); unlocks ONLY via the "Tap to open Harbor" button (root stops pointer/touch/keydown). Kiosk **points reset** (`rpc_kiosk_reset_points`, anon-granted, logs a balancing 'reset' row) in the PIN-gated Parent menu. **Activity ledger** `/app/history` (reward_log + check_ins → labels, day-grouped).
- **Shipped: fairness/verification system** (anti-arguing). Per-chore **`requires_approval`** (migration 0028): tapping a flagged chore on the wall needs the **parent PIN** (reuses `ParentGate`, now with title/subtitle props) before it completes — only gates when a PIN is set (else verifyPin auto-passes = fake gate); ShieldCheck badge; parent toggle on create/edit; routine steps unaffected. Plus a **cooperative reframe**: removed the screensaver "Star leaders" ranking → "Family teamwork" shared-progress panel ("the whole family did it!"). Also: kiosk **confetti + floating +N** on completion (`Confetti.tsx`); **auto-update** SW (reload on new build, throttled) + periodic full reconcile — fixes stale always-on walls.
- **Shipped: Calm Corner (supportive timeout + behavior tracking).** New `corners` table (migration 0029: household/child, reason, feeling, duration_minutes 1-60, started_at/ended_at, status active|done|ended, `plan` jsonb, `report` text, soft-delete; RLS `is_admin() OR child_is_mine`; partial unique `corners_one_active`; rides a rebuilt `kiosk_snapshot` 'corners' key — full pull = active AND not-yet-elapsed via `make_interval(mins=>duration_minutes)`, delta by updated_at so a closed corner syncs away). Parent (child page `CornerCard`): "send to the calm corner" with a **reason** + optional feeling/minutes → `startCorner` (age-based duration ~1 min/yr clamp 2-12, or override) generates a **gentle AI plan** (`lib/ai/corner.ts buildCornerPlan`, Haiku, best-effort → `DEFAULT_CORNER_PLAN`); live countdown; `endCorner`; **AI reflection** (`generateCornerReport`, parent-facing, weaves in past corners). Wall (`CornerTimer.tsx`): soothing countdown + numbered steps + reminder + encouragement, **read aloud once**, "welcome back 💚" + confetti at zero; Home chip "💜 In the calm corner". AI is **strictly non-shaming/non-diagnosing**, age-appropriate (COACH_SYSTEM = co-regulation). Key never reaches the wall (only plan/report text rides the snapshot).
- **Shipped: encouraging voice.** `cheer()` in `lib/kiosk/feedback.ts` (random "Way to go / You did it / Superstar…"); ChildView + ChoresBoard speak `"<cheer>! <task> done!"` on completion; ChildView greets the kid by name on screen-open (read-aloud only, suppressed during a corner).
- **On "subliminal" programming for kids:** declined covert manipulation; built **overt, parent-visible** positive reinforcement instead (screensaver affirmations panel + corner encouragement + greetings). The visible version is ethical and works better; this is the standing approach.
- **Process gotcha:** never pipe `npm run build` to `| tail` when gating a deploy on `&&` — the pipe's exit code is tail's (0), so a failed build still "passes." Use `npm run build > log 2>&1; echo $?` and check the code. (A `state.pinHash` non-null-narrowing error slipped through this way → broken commit + Vercel rejected the deploy; fixed with `state?.pinHash`.)
- **Roadmap the user still wants (not yet built):** **streaks** ("🔥 N days in a row") — the next big dopamine mechanic (needs completion-history plumbing); a visual **per-activity day schedule / "what should I be doing now"** clock for pre-readers (e.g. "brush teeth at 7:30" with a now-marker — needs per-routine-step times); AI "teaching/learning mode" for kids.
- **Verification note:** the kiosk is a PWA with a **service worker that caches the app shell** — the dev preview serves stale kiosk bundles after edits. To verify kiosk changes in preview, unregister SW + clear caches (`caches.keys()→delete`) then reload, OR trust the production build + DB. Keep every feature cost-careful (Haiku, structured outputs, cache once/day). NEVER expose the key to the wall — kiosk AI always goes through a device-validated server route. An adversarial AI security review (key-leak/authz/RLS/cost) is the standard before shipping AI changes.
